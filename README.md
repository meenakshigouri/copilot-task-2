## About This Repository

This repository serves as a **project management knowledge hub** for OctoAcme, designed to demonstrate how to use **GitHub Copilot Spaces** to centralize, democratize, and maintain organizational knowledge.

### 🎯 Purpose

- **Centralize Knowledge**: Consolidate scattered project management processes and documentation
- **Scale Expertise**: Convert tacit knowledge into searchable, versioned, and accessible artifacts
- **Democratize Access**: Give all team members equal access to processes, decisions, and rationale
- **Accelerate Onboarding**: Reduce single-person dependency risk and enable consistent project execution
- **Enable Collaboration**: Connect Copilot Spaces with this repository to extract, refine, and standardize workflows

## 🚀 Getting Started

### Prerequisites
- Access to GitHub Copilot Spaces
- A GitHub Copilot plan (with or without premium features)
- This repository cloned or forked to your account

### Quick Start Steps

1. **Create a Copilot Space** - Navigate to [GitHub Copilot Spaces](https://github.com/copilot/spaces)
2. **Name your Space** - "OctoAcme Project Management Hub"
3. **Add Repository** - Connect this repository as a knowledge source
4. **Select Folders** - Include `docs/` and `.github/ISSUE_TEMPLATE/`
5. **Start Contributing** - Use the Space to generate and refine documentation

For detailed step-by-step instructions, see the [Exercise Issue #1](https://github.com/meenakshigouri/copilot-task-2/issues/1)

## 💡 How This Works

### Copilot Space Workflow

1. **Connect Repository** - The Copilot Space indexes documentation and issue templates
2. **Ask Questions** - Query the Space about project management processes
3. **Generate Content** - Copilot drafts documentation based on your needs
4. **Create Issues** - Propose documentation updates through the Space
5. **Iterate** - Refine and finalize documentation in the repository

### Usage Pattern

```
Developer → Asks Copilot Space → Drafts content → Creates Issue → Merged PR → Updated docs
```

## 📊 Content Guidelines

### Documentation Best Practices

- **Clarity**: Write for someone unfamiliar with OctoAcme processes
- **Completeness**: Include context, examples, and edge cases
- **Consistency**: Follow established naming conventions and formatting
- **Versioning**: Track significant changes through git commits
- **Accessibility**: Use plain language and avoid jargon where possible

### Contributing Documentation

When adding or updating documentation:

1. Create an issue using the appropriate template from `.github/ISSUE_TEMPLATE/`
2. Reference the specific process or workflow being documented
3. Include links to related documentation
4. Submit a pull request with changes
5. Ensure all references are updated

## 🤖 Using Copilot Spaces

### Benefits for Team Knowledge

| Benefit | How It Works |
|---------|-------------|
| **Centralized Knowledge** | All docs indexed in one Space |
| **Quick Discovery** | Search across all documentation in natural language |
| **Consistent Updates** | Changes tracked through issues and PRs |
| **Team Onboarding** | New members query the Space to learn processes |
| **Knowledge Capture** | Convert informal discussions into documented processes |

### Important Notes

- Each prompt in a Copilot Space counts toward your usage quota
- Use **0x models** or **Auto** if available to avoid premium request consumption
- Free tier users: monitor your monthly chat limit
- Premium users: be aware of premium request multipliers

For more details, see [GitHub Copilot Plans](https://docs.github.com/en/copilot/get-started/plans)
